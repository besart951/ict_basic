# Phase 1: Git – Versionskontrolle mit VS Code

In dieser Phase lernst du, wie du deine Projekte sicher speicherst und mit anderen teilst.
Du benutzt dafür die **Quellcodeverwaltung** direkt in VS Code – kein Terminal nötig!

## Was ist Git?

Git ist wie eine Zeitmaschine für deinen Code. Du kannst:
- Änderungen speichern (**Commit**)
- Änderungen hochladen (**Push**)
- Änderungen herunterladen (**Pull**)

## Lernziele

- Ein Repository auf GitHub erstellen
- Änderungen committen
- Änderungen pushen und pullen

Arbeite die Dateien in dieser Reihenfolge durch:
1. `01_git_basics.md`
2. `02_first_commit.md`
3. Übungen in `uebung/`
4. Herausforderung in `herausforderung/`
